## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap within badge, inline chip padding |
| sm | 8px | Gap between badge and text in match card |
| md | 16px | Card padding (`p-4`), section horizontal padding, form field spacing |
| lg | 24px | Section vertical gap, space between match card groups |
| xl | 32px | Page top padding above list |
| 2xl | 48px | Empty state vertical centering padding (`py-16` = 64px, acceptable as 4-multiple) |
| 3xl | 64px | Bottom nav height (fixed, already established) |

Exceptions:
- Bottom nav: `calc(64px + env(safe-area-inset-bottom))` — inherited from Phase 1, do not change.
- Min touch target: 44px min-height on all interactive cards and buttons (already enforced via `min-h-[44px]` pattern from Phase 1).
- FAB (floating action button for add match): 56px diameter, fixed bottom-right, above bottom nav with `bottom: calc(80px + env(safe-area-inset-bottom))`.
- CSV preview table rows: 48px min-height for thumb-scroll comfort.

## Typography

Inherit exactly from Phase 1 — do not introduce new sizes or weights.

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 16px | 400 normal | 1.5 | `text-base font-normal` | Match card secondary info, form labels, CSV preview cell text |
| Label | 14px | 400 normal | 1.4 | `text-sm font-normal` | Muted secondary info (hora, fecha_nro), badge text, helper text |
| Heading | 20px | 600 semibold | 1.3 | `text-xl font-semibold` | Page heading ("Fixture"), section headers (division groups in admin view) |
| Card primary | 16px | 600 semibold | 1.25 | `text-base font-semibold` | Rival name in match card (primary text, most important at a glance) |

Font: `var(--font-sans)` (Inter). No additional font declarations.

Score display exception: The own/opponent score pair on match detail uses `text-4xl font-bold` (36px, weight 700) — this is a single-use display element for instant readability at arm's length on a sports field. Not a reusable typography role.

## Color

All values use existing CSS custom properties from `globals.css`. Do not hardcode hex values.

| Role | CSS Variable | Approximate Value | Usage |
|------|-------------|-------------------|-------|
| Dominant (60%) | `var(--background)` | `oklch(0.12 0 0)` — near-black | Page background, screen fill |
| Secondary (30%) | `var(--card)` | `oklch(0.18 0 0)` — dark card | Match cards, form sections, CSV preview table container |
| Accent (10%) | `var(--primary)` | `oklch(0.40 0.12 142)` — rugby green | Reserved elements listed below |
| Muted surface | `var(--secondary)` / `var(--muted)` | `oklch(0.22 0 0)` | Section dividers, disabled states |
| Destructive | `var(--destructive)` | `oklch(0.65 0.20 22)` — red-orange | Delete match button and confirmation dialog only |

Accent (`var(--primary)` / green) is reserved **only** for:
1. FAB background (add match button)
2. Active bottom nav tab icon (Fixture tab, inherited pattern)
3. "Confirmar import" button (primary action in CSV flow)
4. "Guardar resultado" button (primary action on match detail)
5. Position badge circles (inherited from Phase 1 — `PlayerPositionBadge` pattern)

Result badge colors — implemented as Tailwind classes, NOT as `var(--primary)`:

| State | Badge color | Tailwind classes |
|-------|-------------|-----------------|
| Ganado (won) | Green | `bg-green-700 text-green-100` |
| Perdido (lost) | Red | `bg-red-700 text-red-100` |
| Pendiente (no result) | Muted grey | `bg-muted text-muted-foreground` |

Subequipo chip (M17 "A" badge):

| Element | Tailwind classes |
|---------|-----------------|
| Subequipo chip | `bg-secondary text-secondary-foreground text-xs font-semibold px-1.5 py-0.5 rounded` |

"Jugado sin resultado" state (match played, date passed, no score entered):
- Same grey `Pendiente` badge
- Secondary line in card: `"Sin resultado cargado"` in `text-sm text-muted-foreground`
- Distinguishes from upcoming (which shows no secondary status line)

## Component Inventory

### MatchCard (new component)

Layout: `Card > CardContent` with `p-4 flex flex-col gap-1 min-h-[72px]`

Structure (top to bottom, left to right):

```
Row 1: [fecha "Sáb 12/04"] ········ [Fecha #1] [subequipo chip "A"?] [result badge]
Row 2: [rival name — bold, truncate]
Row 3: [L/V · hora] ················ [ChevronRight icon]
```

Implementation detail:
- Row 1: `flex items-center justify-between gap-2`
- Row 2: `text-base font-semibold leading-tight truncate` for rival name
- Row 3: `flex items-center justify-between text-sm text-muted-foreground`
- L/V displayed as: "Local" or "Visitante" (full word, not abbreviation)
- `ChevronRight` from lucide-react, `h-5 w-5 text-muted-foreground shrink-0` — same as SessionCard pattern
- Entire card is a `Link` wrapping `href="/fixture/[matchId]"` — same pattern as PlayerCard and SessionCard

### Division Section Header (admin "All Divisions" view)

```
<h2 class="text-xl font-semibold px-4 pt-6 pb-2">M19</h2>
```

Renders above each division's match group when admin selects "Todas las divisiones".

### ResultBadge (new component)

Props: `result: 'won' | 'lost' | 'pending'`

```
won:     <Badge className="bg-green-700 text-green-100">Ganado</Badge>
lost:    <Badge className="bg-red-700 text-red-100">Perdido</Badge>
pending: <Badge variant="secondary">Pendiente</Badge>
```

When score is available, badge shows score inline: "Ganado 24-17" / "Perdido 10-21".

### Match Detail Screen

Route: `/fixture/[matchId]/page.tsx` — full scrollable page, not a modal (D-05, D-06).

Layout sections (top to bottom):

1. **Back navigation**: `<button>` with `ChevronLeft` + "Fixture" label, `h-11` tap target, `text-base font-semibold`
2. **Match header card**: date, rival (large), L/V, hora, venue if available, subequipo chip
3. **Score section**: Own score / Rival score as two `Input type="number"` fields side by side, `h-14 text-4xl font-bold text-center`. "Guardar resultado" Button below — full width, `h-12 text-base`.
4. **Scoring detail section** (collapsible, default collapsed for played matches with score already set; expanded for new score entry):
   - Section heading: "Detalles del resultado" with expand/collapse toggle (ChevronDown/ChevronUp icon)
   - Content: tabbed or stacked lists for Tries, Conversiones, Penales, Drop Goals, Tarjetas
   - Each event: player Select (division roster) + event type (for cards: amarilla/roja)
   - Add row button: `text-sm text-primary` link-style button "+ Agregar try / conversión / etc."
   - All optional — section can remain empty (D-08)
5. **Edit / Delete actions** (admin only): rendered at bottom of page, separated by `<Separator />`

Collapsible animation: CSS `transition-all duration-200` on max-height — no animation library needed.

### Match Form (add/edit)

Rendered in a `Dialog` (D-04, reuses existing Dialog pattern from DeletePlayerDialog).

Fields:
- `fecha` — `Input type="date"` 
- `rival` — `Input type="text"` placeholder "Nombre del rival"
- `local_visitante` — `Select` with options "Local" / "Visitante"
- `hora` — `Input type="time"`
- `division` — `Select` (editable for admin, read-only display for coach — shows active division)
- `subequipo` — `Input type="text"` placeholder "A, B… (opcional)" — only visible for M17, or always visible with hint

Dialog footer: Cancel (ghost variant) + "Guardar partido" (default variant, primary green).

### CSV Import UI

Route: `/admin/fixture-import` or within `/fixture` page admin section — admin-only (D-13).

Flow: upload → preview → confirm (D-11).

Step 1 — Upload:
- `Input type="file" accept=".csv"` inside a Card
- Heading: "Importar fixture URBA" (`text-xl font-semibold`)
- Helper text: `"Formato CSV con columnas: division, fecha_nro, fecha, equipo, local_visitante, rival, hora"` in `text-sm text-muted-foreground`
- "Subir archivo" Button — primary, full-width, `h-12`

Step 2 — Preview:
- Table with columns: División, Fecha #, Fecha, Equipo, L/V, Rival, Hora
- `Table` shadcn component, compact rows (`h-12`), `text-sm`
- Error rows (parse failures) highlighted: `bg-destructive/20 text-destructive` row, error message in last column
- Row count summary above table: `"Se encontraron 60 partidos. N errores."` in `text-sm text-muted-foreground`
- If errors > 0: warning text before confirm button, confirm button still enabled (partial import allowed)

Step 3 — Confirm:
- "Confirmar import" Button — primary, full-width, `h-12` — triggers truncate + insert
- "Cancelar" Button — ghost variant, full-width — returns to upload step
- Warning text: `"Esto reemplazará todos los partidos URBA importados. Los partidos manuales se preservan."` in `text-sm text-amber-400`

Loading states: all buttons show `Skeleton` or disabled + spinner icon during async operations.

## Accessibility

- All interactive cards: `focus-visible:ring-2 focus-visible:ring-[color:var(--ring)] rounded-lg` (same as PlayerCard pattern)
- FAB: `aria-label="Agregar partido"`
- Result badge: `aria-label="Resultado: Ganado 24-17"` (includes score in label)
- Score inputs: `aria-label="Puntos Virreyes"` / `aria-label="Puntos [rival]"`
- CSV file input: `aria-label="Archivo CSV del fixture"`
- Collapsible scoring detail: `aria-expanded` attribute toggled on trigger button
- Minimum tap target 44px on all interactive elements (enforced via `min-h-[44px]` or explicit `h-11`/`h-12` classes)

---

## Navigation Pattern

- Fixture tab → `/fixture` — list of matches for active division
- Match card tap → `/fixture/[matchId]` — match detail (push navigation)
- Back from detail → back button to `/fixture` (Next.js `router.back()` or `Link href="/fixture"`)
- Admin "Todas las divisiones" → same `/fixture` route, DivisionSelector emits `null` division ID → query fetches all juvenile divisions, grouped render
- CSV import → `/admin/fixture-import` (admin-only route, linked from admin tab or from fixture empty state for admin)
- Add match → FAB opens `Dialog` (no route change) — same pattern as no page-push for simple forms
